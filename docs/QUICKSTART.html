Job Application MCP — Quick Start

v0.4.0  ·  one-click setup  ·  local-first  ·  github.com/firyomaefx/job-application-mcp

You don't need to configure anything by hand. The app detects what's available and sets itself up with a single click.

① Install & open Download the Windows installer (NSIS .exe or .msix) from the GitHub Releases page and run it — or, for the command line, git clone the repo, npm install, npm run build. Open the app; the local bridge starts automatically.
First run on Windows: the installer is currently unsigned, so SmartScreen warns. Choose More info → Run anyway. It's the genuine build — the warning is reputation, not malware.
② Click “Auto-configure & start” On the Setup & Status card at the top of the dashboard, click the big button. The app:
  1. probes for a local Ollama model (fully offline, free, no key);
  2. if Ollama isn't running, checks whether you've previously set an AI key;
  3. otherwise falls back to the built-in Mock heuristic (offline, no key);
  4. applies the best option and confirms the bridge is up.
A green checklist appears showing the state of Bridge · AI · Profile · CV.
Bridge: ● running :8787 AI: ● Ollama (local) · up [ Mock ] [ ●Ollama ] [ My key ] Profile: ● set CV: ○ none — [ Import CV ]
③ Change anything with one click The same card has one-click chips to switch AI provider (Mock / Ollama / My key) — applied live, no restart. The collapsible AI settings form lets you set the model, base URL, and API key (your own OpenAI/Anthropic key, stored locally, masked, never uploaded). Use Import CV to parse a local PDF/DOCX/TXT, and Set up profile to add your name/skills.

Then what?

  • From an AI client (Claude Desktop / Claude Code / Cursor): register the MCP server and just talk — “parse my CV, analyze this job, tailor my CV for it.” The system_check tool reports the same status the dashboard shows.
  • From the browser: the Chrome extension detects job postings on LinkedIn/Indeed/Lever/Greenhouse/Workday/SmartRecruiters and imports them to your local inbox with one click.
  • Submission is always manual: the app prepares and previews; you submit on the employer's site, then record it. Nothing is ever auto-submitted.

Everything stays on your machine

CVs, applications, reminders, and your AI key live under JOB_MCP_DATA_DIR. The bridge binds 127.0.0.1 only. The only outbound calls are to the AI provider you chose (your key or local Ollama) and, on packaged builds, a GitHub update check. No CV text or PII is ever uploaded or logged.

Full tool reference (40+ tools) and config details: see the README at github.com/firyomaefx/job-application-mcp.
